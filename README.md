# SupplyFind — Система поиска поставщиков для HoReCa

## Описание
Платформа для автоматизированного поиска, глубокого анализа и верификации B2B-поставщиков продуктов питания. Сервис использует поисковые AI-модели и автоматизированный парсинг для сбора контактов, сертификатов и прайс-листов, идеально подходя для сетей «Жизнь Март» и «Сушкофф».

## Архитектура
```text
Flutter Web -> FastAPI -> [Redis Cache] -> PostgreSQL
                      -> Tavily API
                      -> Deep Parser (httpx + BS4)
                      -> ФНС API (верификация)
```

## Быстрый старт
1. Скопируйте файл переменных окружения:
   `cp backend/.env.example backend/.env`
2. Заполните `TAVILY_API_KEY` в файле `.env`.
3. Запустите стек через Docker Compose:
   `docker-compose up --build`
4. Откройте интерфейс в браузере: `http://localhost:8080`

## Переменные окружения (.env)
```env
DATABASE_URL=postgresql://user:password@db:5432/food_supplier_db
REDIS_URL=redis://redis:6379
TAVILY_API_KEY=your_key_here
FNS_API_KEY=demo
```

## API Endpoints

| Метод | Путь | Описание | Параметры |
|-------|------|----------|-----------|
| `POST` | `/api/suppliers/search` | Умный поиск поставщиков с учетом фильтров и скоринга | `category`, `city`, `verified_only`, `has_certificate`, `min_score` |
| `GET` | `/suppliers/{id}` | Получение детальной карточки поставщика | `supplier_id` |
| `PATCH` | `/suppliers/{id}` | Обновление данных поставщика вручную | Тело запроса `SupplierUpdate` |
| `POST` | `/suppliers/compare` | AI-скоринг и сравнение нескольких компаний | Список `supplier_ids` |
| `POST` | `/suppliers/{id}/hide` | Добавление поставщика в черный список | `supplier_id` |
| `POST` | `/suppliers/discover/all` | Глобальный фоновый скан всего рынка | Нет |

## Как работает поиск
1. **Первичный поиск:** Система делает запрос к поисковому AI (Tavily), исключая мусорные сайты и агрегаторы (2GIS, Авито, Контур).
2. **Deep Parsing:** Бэкенд заходит на каждый найденный официальный сайт, обходит разделы ("Контакты", "О компании") и с помощью регулярных выражений извлекает уникальные телефоны, email и ИНН.
3. **Верификация (ФНС):** Если найден ИНН, система обращается к API ФНС (через DaData/api-fns.ru), чтобы проверить, действует ли компания, как долго она на рынке и кто её директор.
4. **Скоринг:** Применяется бальная система (0-100). Дополнительные баллы даются за верификацию, наличие сертификатов (ГОСТ, Халяль) и корпоративные контакты.
5. **Кэширование:** Все результаты и история сохраняются в PostgreSQL и кэшируются в Redis на 1 час для экономии API квот.

## Структура проекта
- `/backend/app/` — Исходный код FastAPI
  - `/api/` и `main.py` — Роутеры и REST-контроллеры
  - `/services/` — Внутренняя бизнес-логика (парсинг, верификация, скоринг)
  - `/db/` — Модели SQLAlchemy, подключение к PostgreSQL и тестовые данные (Seed)
- `/frontend/` — Flutter Web приложение
  - `/lib/screens/` — Пользовательские экраны (карточки, сравнение)
  - `/lib/widgets/` — Компоненты UI (вкл. мини-игру)
  - `/lib/services/` — Взаимодействие с API бэкенда
- `docker-compose.yml` — Инфраструктурный манифест (DB, Cache, API, UI)
