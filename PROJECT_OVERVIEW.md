# Техническое описание проекта: Food Supplier Service (для LLM)

## 1. Концепция
Сервис для автоматизации поиска и сравнения поставщиков продуктов питания для сетей «Жизнь Март» и «Сушкофф». Система использует AI-агентов для сбора данных из интернета в режиме реального времени.

## 2. Технологический стек
- **Backend:** Python 3.9, FastAPI (REST API).
- **Database:** PostgreSQL (SQLAlchemy ORM).
- **Frontend:** Flutter (Web/Desktop/Mobile).
- **AI/Search:** Tavily AI API (для глубокого поиска), DuckDuckGo (fallback), встроенная логика очистки данных на Regex.
- **Инфраструктура:** Docker, Docker Compose (3 контейнера: db, backend, frontend).

## 3. Текущий функционал
1. **AI Discovery (Агентский поиск):** 
   - Поиск через Tavily по категориям и городам РФ.
   - Очистка названий от SEO-мусора.
   - Сбор только реальных данных (Name, Snippet, URL).
   - Пакетный поиск (Batch Discovery) по всем категориям.
2. **Управление данными:**
   - Ручное добавление поставщиков.
   - Редактирование всех полей (ИНН, сайт, контакты, рейтинг).
   - Система «черного списка» (скрытие мусорных компаний с функцией Undo).
3. **Аналитика:**
   - Таблица сравнения нескольких поставщиков.
   - Интеллектуальный отчет-резюме (генерация вывода на основе рейтинга и наличия сертификатов).
4. **Интерактив:**
   - Кликабельный ИНН (переход в Контур.Фокус).
   - Интерактивные контакты (tel:, mailto:, url).
   - Мини-игра в окне ожидания AI-поиска.
5. **UI/UX:** 
   - Material 3 дизайн.
   - Темная/Светлая темы.
   - Локализация (RU/EN).

## 4. Запрос для ТЗ (Инструкция для LLM)
Используй это описание, чтобы составить ТЗ для следующего этапа разработки. Сосредоточься на:
- Повышении качества извлечения ИНН и контактов через глубокий парсинг страниц (Firecrawl).
- Реализации реального анализа PDF/Excel прайс-листов.
- Автоматизации RFQ (отправка реальных запросов).
- Интеграции с API DaData для верификации контрагентов по ЕГРЮЛ.
- Не используй эмодзи в ответах.
